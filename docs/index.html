What It Does

⚡

Token Usage Ingestion

POST /api/v1/events

Accepts CloudEvents v1.0 with per-request token breakdown: input, output, cached read, cache write, reasoning tokens.

🛡

Balance Checks

GET /api/v1/customers/{id}/entitlements/{key}/value

Real-time quota enforcement. The plugin checks balance before processing each inference request.

📊

Usage Dashboard

GET /dashboard

Built-in dashboard with per-user, per-model token breakdown and cache-aware cost calculation.

Supported Metering Providers

The external-metering plugin emits CloudEvents v1.0 — any backend that accepts this format works:

EnvironmentBackendStatus
DevelopmentThis service — drop-in simulator with PostgreSQL + dashboard✅ Available
StagingOpenMeter — open-source usage metering✅ Compatible
ProductionMonetize360 — enterprise billing (Verizon)🚧 Planned
CustomAny CloudEvents v1.0 endpoint🔧 Extensible

Token Types Tracked

Cache-aware metering — different token types have different costs:

Token TypeOpenAI FieldAnthropic FieldPricing Impact
● Input (new)prompt_tokensinput_tokensFull input rate
● Outputcompletion_tokensoutput_tokensFull output rate
● Cached readcached_tokenscache_read_input_tokens~10x cheaper
● Cache write—cache_creation_input_tokens~25% more
● Reasoningreasoning_tokens—Output rate

Quick Start

Docker Compose

git clone https://github.com/noyitz/ai-gateway-metering-service
cd ai-gateway-metering-service
docker compose up

Kubernetes

kubectl create namespace metering
kubectl apply -f deploy/ -n metering

Enable in AI Gateway

upstreamBbr:
  bbr:
    plugins:
      - type: external-metering
        name: external-metering
        json:
          meteringURL: "http://metering-service.metering.svc:8080"
          failOpen: true
